# 双轨混合驱动 Android 自动化测试系统架构

> 本文档描述基于 Midscene.js 进行改造的混合驱动系统架构设计。

## 一、设计背景与目标

### 问题分析

当前 Midscene Android 自动化存在两个核心问题：

| 问题 | 表现 | 根因 |
|-----|-----|-----|
| AI推理慢 | 每次操作需 3-5s | 纯视觉方案依赖模型响应 |
| AI规划有限 | 找元素只往一个方向滑 | 缺乏全局感知和变通能力 |

### 设计目标

- **保留优势**：自然语言描述，无需维护选择器
- **解决痛点**：重复操作速度提升 10-100 倍
- **增强能力**：智能导航，双向搜索，系统级操作

### 核心思想

**第一次用 AI 看，之后用结构跑。**

利用 Android 内建的 `uiautomator dump` 命令获取 UI 树结构，构建多级缓存系统，实现快速定位。

## 四、核心组件设计

### 4.1 智能路由器 (SmartRouter)

**职责**: 按优先级选择定位方式，决定是否调用 AI

**路由逻辑**:

```
locate(description):
  1. L1会话缓存 → 命中则直接返回坐标 (~50ms)
  2. L2 UI树解析 → 文本/resource-id匹配 (~300ms)
  3. L3 页面快照 → 页面指纹匹配 (~100ms)
  4. L4 AI视觉 → 截图+模型推理 (~3-5s)
  
  每次成功定位 → 回写所有缓存层
```

**关键方法**:

| 方法 | 功能 |
|-----|-----|
| `locate(description)` | 核心路由入口 |
| `quickVerify(entry)` | 快速验证缓存有效性 |
| `learnFromAI(description, result, uiTree)` | AI结果学习，提取结构化定位信息 |

### 4.2 UI树解析器 (UITreeParser)

**职责**: 获取并解析 Android UI 结构

**技术实现**:

```bash
# 获取 UI 树
adb shell uiautomator dump /sdcard/ui.xml
adb pull /sdcard/ui.xml

# 解析得到:
- resource-id
- text / content-desc
- bounds (精确坐标)
- clickable / scrollable / enabled 属性
```

**关键方法**:

| 方法 | 功能 |
|-----|-----|
| `dump()` | 获取当前屏幕 UI 树 |
| `findByText(tree, text)` | 文本匹配查找 |
| `findByResourceId(tree, id)` | resource-id 匹配 |
| `findElementAtCoords(tree, bounds)` | 坐标反查元素 |
| `generateXPath(node)` | 生成 xpath 路径 |

### 4.3 导航策略引擎 (NavigationStrategy)

**职责**: 解决"只往一个方向滑"的问题

**策略优先级**:

```
smartFind(description):
  1. 判断是否找应用 → 使用系统搜索 (最快)
  2. 双向滑动搜索 → 先下后上，检测页面卡住
  3. 智能回退 → 页面无变化时换方向
```

**核心能力**:

| 能力 | 实现 |
|-----|-----|
| 系统搜索 | `monkey -p packageName` 或 `KEYCODE_SEARCH` |
| 双向滑动 | 往下滑检测，卡住则往上滑 |
| 页面状态检测 | 感知哈希判断页面变化 |

### 4.4 混合缓存管理器 (HybridCacheManager)

**职责**: 多级缓存的生命周期管理

**缓存层级**:

| 层级 | 存储 | TTL | 用途 |
|-----|-----|-----|-----|
| L1 | 内存 | 无 | 当前测试会话 |
| L2 | 内存 | 1分钟 | 页面快照复用 |
| L3 | 内存 | 24小时 | UI树结构 |
| L4 | 文件 | 持久 | 跨测试复用 |

**缓存记录结构**:

```typescript
{
  description: "登录按钮",
  pageFingerprint: "hash_abc123",
  locateMethods: [
    { type: "text", value: "登录", bounds: [100,200,200,250] },
    { type: "resource-id", value: "com.app:id/btn_login" },
    { type: "xpath", value: "//node[@text='登录']" },
    { type: "visual", modelResponse: {...} }  // AI兜底
  ],
  successRate: 0.95,
  lastUsed: 1709123456789
}
```

```yaml

  - name: 登录流程
    flow:
      - tap: "登录按钮"           # → hybridTap
```

## 六、性能预估对比

| 场景 | 原 Midscene | 混合方案 | 提升 |
|-----|------------|---------|-----|
| 首次定位新元素 | 3-5s | 3-5s | 0% |
| 重复操作相同元素 | 3-5s | 0.05s | **99%** |
| 同页面不同元素 | 3-5s | 0.3s | **90%** |
| 跨页面找元素 | 10-30s (可能失败) | 2-5s | **80%** |
| 找应用启动 | 滑动搜索 30s+ | 系统搜索 1s | **97%** |

## 九、风险与应对

| 风险 | 影响 | 应对方案 |
|-----|-----|---------|
| uiautomator dump 失败 | L2 不可用 | 回退到 AI |
| UI树解析耗时 | 性能下降 | 短时缓存 + 增量解析 |
| 页面动态变化 | 缓存失效 | 页面指纹校验 + 自动重新学习 |
| 特殊字符匹配失败 | 定位失败 | 多种匹配策略 + AI兜底 |

## 十、总结

本架构设计的核心是 **"学习+执行"双轨模式**：

1. **学习阶段**: AI 视觉分析 UI，提取结构化定位信息
2. **执行阶段**: 结构化定位优先，AI 视觉兜底

这种模式兼顾了：
- **易用性**: 自然语言描述，无需维护选择器
- **高性能**: 重复操作跳过 AI，速度提升 10-100 倍
- **可靠性**: 多级缓存 + 多种定位方式，自动回退机制

最终实现 **首次慢、后续快** 的理想效果。
